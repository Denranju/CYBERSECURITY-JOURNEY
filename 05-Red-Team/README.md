# 📚 Topics Covered

## What is Penetration Testing?

Penetration testing is the authorized assessment of computer systems, networks or web applications to identify security vulnerabilities before attackers can exploit them.

Penetration tests help organizations improve their security posture by discovering weaknesses in a controlled and ethical manner.

---

## Ethical Hacking

Ethical hackers perform security assessments with permission from the system owner.

Their objective is to discover vulnerabilities, document findings and recommend security improvements.

---

## Penetration Testing Methodology

A typical penetration test includes:

1. Planning
2. Reconnaissance
3. Enumeration
4. Vulnerability Analysis
5. Exploitation
6. Post-Exploitation
7. Reporting

---

## Reconnaissance

Reconnaissance is the process of collecting information about a target before attempting to identify vulnerabilities.

Examples include:

- DNS enumeration
- Directory discovery
- Website fingerprinting
- Source code analysis

---

## Enumeration

Enumeration involves actively gathering information from target systems.

Examples include:

- Services
- Open ports
- User accounts
- Directories
- Web technologies

---

## HTTP Requests

HTTP is the protocol used for communication between web browsers and web servers.

Common HTTP methods include:

- GET
- POST
- PUT
- DELETE
- OPTIONS
- HEAD

---

## HTTP Response Codes

Examples include:

- 200 OK
- 301 Moved Permanently
- 302 Found
- 403 Forbidden
- 404 Not Found
- 500 Internal Server Error

---

## HTTP Headers

HTTP headers exchange additional information between clients and servers.

Examples include:

- User-Agent
- Referer
- Host
- Cookie
- Authorization

---

## Cookies

Cookies store session information and user preferences inside the browser.

During my practical labs I learned how modifying cookies can change application behaviour and why applications should never trust client-side data.

---

## Web Applications

A web application consists of:

- Client
- Server
- Database

These components communicate using HTTP or HTTPS.

---

## Common Web Vulnerabilities

During my studies I learned about:

- Information Disclosure
- Directory Enumeration
- Exposed Source Code
- Cookie Manipulation
- HTTP Header Manipulation
- Authentication Weaknesses
- Input Validation Issues

---

## Reporting

An important part of penetration testing is documenting findings clearly and providing remediation recommendations.
